Алгоритм создания для маленьких серверов с данными в памяти. Только бэкэнд.

1. Создаём index.js, в нём будет весь код сервера на основе модуля http.
2. Функция-обработчик запросов находится в нём же. Запросы дифференциируются по свойству .url, что позволяет реализовать нехитрый роутинг уровня "if / else if / else if / else".
3. Один роут (обычно '/') отдаёт html-разметку страницы по принципу "all inclusive", и если/когда понадобятся стили, то и CSS в тэге <style>, а если/когда понадобится поведение, то и JS в тэге <script>.
4. Остальные роуты отдают / принимают данные, загружая из / сохраняя в оперативной памяти.
5. Всей логики и кода страницы должно быть очень-очень мало, чтобы было оправданным умещение всего в одном файле index.js (почти никогда не рекомендуется задерживаться в этом варианте проекта).

6. Опубликовать такой проект можно на любом сервере-хостинге, поддерживающем NodeJS. Например на heroku.com.
